	- DevOps ::::
	
		What is DevOps ?
		
		DevOps is Software Development Strategy, which helps build the collaboration between teams like Development Team and Operations Team to achieve Continuous Development, Continuous Integration, Continuous Testing, Continuous Delivery, Continuous Deployment and Continuous Monitoring in a more automated fashion.

		- DevOps Stages :
		
			- Continuous Development :
			
				- Continuous Development is the capability of the Development Team to Continuous to develop the changes and integrate for further testing and implementation.
				- It is used to improve the Developers' Productivity
				
				Role Developers:
					- Create Source Code
					
				Developer :
				
					- Create Source Code 
					- Manual Build, Create Artifacts (binaries - *.war/*.jar/*.exec,*.pkl)
					- Perform manual unit test 
					- Promote the changes to test environment 
					- Notify the Testing Team - Thru Email 
					
				Using DevOps Approach: 

					- Create Source Code 
					- Save the Source Code in a Code Repository(Github)
				
				Automate:
				
					- Build, Create Artifacts (binaries - *.war/*.jar/*.exec,*.pkl)
					- Perform unit test 
					- Promote the changes to test environment 
					- Notify the Testing Team - Thru Email 
					
			
				Tools :
				
					- IDEs - Eclipse based IDEs - Pycharm, VS Code, Visual Studio  
					- GIT, GitHub
					
			
			- Continuous Integration :::
			
				- Continuous Integration is the capability of the Development Team to Continuously integrate the changes for further testing.	

				Automate:
				
					- Build, Create Artifacts (binaries - *.war/*.jar/*.exec,*.pkl)
					- Perform unit test 
					- Promote the changes to test environment 
					- Notify the Testing Team - Thru Email

				Tools :
					- IDEs - Eclipse based IDEs - Pycharm, VS Code, Visual Studio  
					- GIT, GitHub
					- Jenkins, Github-Actions, Az-Pipelines, gitlab-ci
					- Docker, Kubernetes, Ansible
					
			
			- Continuous Testing ::
				- Continuous Testing is the capability of the Test Team to Continuously test the changes.
				
				Tools :
					- Jenkins, Github-Actions, Az-Pipelines, gitlab-ci
					- Selenium
					
			
			
			- Continuous Delivery / Deployment ::
			
				- Continuous Delivery 
					- It is used to release the changes to Production Environment with manual approvals
					- This leads to Application Downtime.

					Eg.: 
					Typical - Core Banking Application:
					
						Online Banking - 
						
							Downtime - Release Window 
								- (4 - 6hrs)
								
								- Production 
								
								- (Fix / Revert)
							
							
				- Continuous Deployment
					- It is used to release the changes to Production Environment without any manual approvals
					- This can be achieved only with the Micro-Service based Application Architecture
					- We can implement the changes to Production without any downtime(ZERO-Downtime)

					Eg.: 
					- Netflix/Facebook/amazon ::		
					
					
					
				www.amazon.com ::
				
					Sign-Up 		--> Micro-Service1 -> Developer1 -> code,test,release to prod
					Sign-In 		--> Micro-Service2
					Search 
					Add-to-cart
					Place Order
					Payment 
					Confirm 
					Track 
			
			- Application Architecture :
			
				- Monolith Application Architecture
					- It is tightly coupled by default 
					- All the dependencies are tightly coupled together and deployed a a whole appplication.
					- With Monolith Application Architecture we can achieve only Continuous Delivery
				
				
				- Micro-Service Based Application Architecture
					- It is loosely Coupled Application Architecture
					- Each function/Module is called a Micro-Service
					- Each Micro-Service can be independently developed, test, and deployed to Production Environments
					- With Micro-Service based Application Architecture we can achieve Continuous Deployment
			
			
				Tools ::
				
					- Jenkins,Github-Actions,gitlab-ci 
					- docker,kubernetes,Ansible
